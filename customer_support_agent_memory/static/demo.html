Customer Support Widget

An embeddable AI chat widget with persistent memory. One script tag — your users get support that remembers them.

Quick Start — Embed the Widget

Add a single script tag to your site and the chat bubble appears automatically.

<script src="https://your-api.com/static/widget.js" data-domain-id="YOUR_DOMAIN_ID"></script>
⚠️ You need a Domain ID from the registration step below. Without it the widget will not load.

How to get your Domain ID

  1. Open Swagger UI → /docs
  2. Find POST /register-domain and call it with your site hostname — e.g. {"domain_name": "example.com"} (no scheme, no path)
  3. Copy the domain_id UUID from the response
  4. Paste it into the data-domain-id attribute above

Widget Controls (this page)

Configuration Options

All options can be set via data-* attributes on the script tag.

<script src="/static/widget.js" data-domain-id="YOUR_DOMAIN_ID" data-api-url="https://your-api.com" data-primary-color="#0061ff" data-widget-title="Help & Support" data-position="bottom-right"></script>

Security Model

The widget uses a domain ID (a server-issued UUID), not a secret key. There are no credentials in the browser.

  • The widget sends X-Domain-ID: <uuid> with every request
  • The server looks up the UUID in the registered_domains table
  • Per-domain memory isolation — users on site A never see memories from site B
  • Invalid or missing domain ID returns 400/401 immediately
✓ No secrets in browser ✓ Per-domain isolation ✓ Per-user memory scoping

How It Works

  1. Widget opens → creates a session tied to this user (stored in localStorage for continuity)
  2. User asks a question → sent to POST /ask with the domain ID and session ID
  3. The server retrieves the DigitalOcean Gradient AI agent for this domain
  4. Memori recalls relevant facts from this user's past conversations and injects them as context
  5. The LLM (Llama via DigitalOcean) answers using knowledge base content + recalled user context
  6. Memori stores new facts from the answer in the background for future recall
ℹ️ DigitalOcean handles what the site does (RAG over your content). Memori handles who the user is (persistent cross-session memory).
